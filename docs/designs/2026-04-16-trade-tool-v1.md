## 컨텍스트

현재 사이트는 "한국 자율주행택시 산업을 한눈에 보여주는 대시보드"라는
모호한 포지셔닝으로 운영 중. 5개 페이지 (홈/지도/타임라인/글로벌비교/기업상세),
580+ 뉴스 자동 크롤링, 9개 기업 데이터, 29개 시범운행지구 데이터 보유.

기술적으로는 잘 만들어졌으나, **"누구를 위한 도구인가"가 모호**하여
재방문 동기가 약한 상태. 또한 라이브 사이트에 P0 버그 존재 (지도 렌더링 실패,
일부 차트 빈 영역).

## 1순위 사용자 결정

**업계 관계자 (PM/엔지니어)** — 자율주행 사업 부서, 모빌리티 스타트업,
부품·SW 협력사, 정책기관 담당자.

이 사용자의 행동 양식:
- 산업 기본 지식 있음 → 위키성 설명 불필요
- **신선도가 가치** — 뉴스가 12시간 늦으면 슬랙에 이미 돌고 있음
- **본인 회사 vs 경쟁사** 비교가 일상 — 글로벌보다 국내 1:1 비교가 더 가치
- 규제 변화 = 직접 사업 영향 → 푸시/알림이 핵심
- 출퇴근/회의 중 모바일 확인 → 모바일 UX 중요
- 신뢰성 = 직장 내 인용 가능성 → 출처·날짜·근거 명확해야

핵심 변화 3가지:
1. **수동 → 능동 전달**: 사용자가 오는 게 아니라, 내가 사용자에게 간다 (RSS/뉴스레터)
2. **읽기 전용 → 도구**: 데이터 보여주는 곳이 아니라, 일에 쓸 수 있는 자료 (export, 비교, 북마크)
3. **잡지 → 트래커**: "최근 뉴스 목록"이 아니라 "지난 7일간 X 기업이 한 일" 단위로 묶음

## Implementation Approach

**Approach B: P0 버그 수리 + Killer Feature Bundle 1개**

다른 후보:
- Approach A (Boil the Lake — 12개월 비전 한 번에) → 1인 유지 부담 큼, 일부 기능 검증 안 된 채 출시 위험
- Approach C (버그만 고치고 끝) → 전략적 진전 zero, 재방문 동기 미해결

선택 사유:
- L 사이즈 = CC+gstack 환경에서 1.5일 → 압축 효과 큼
- "좋은 도구는 한 가지를 끝까지 완성" (Rams)
- 객관적 PMF 지표 측정 가능 (구독자 수, 비교 페이지 뷰)

## Edge Cases (검증 항목)

### B. 1:1 비교
- 같은 회사 선택 (`?a=swm&b=swm`) → 리다이렉트
- 존재 안 하는 slug → 404 + 회사 선택 UI
- slug 한글/영문 정규화 → companies.json `slug` 필드로 정규화
- OG 이미지 동적 생성 (두 회사명 포함)

### D. RSS / CSV / JSON
- 뉴스 580+건 RSS → 최근 50건만 (RSS 표준)
- CSV escaping (RFC 4180, 콤마/줄바꿈 안전)
- JSON 캐싱 (ISR revalidate=3600)

### 5. 북마크
- localStorage 비활성화 (사파리 시크릿) → in-memory fallback + "저장 안 됨" 토스트
- 다른 기기 동기화 안 됨 → 명시적 안내
- 회사 ID 변경 시 stale bookmark → ID 검증 후 silent skip
