### `pickRepresentative(items)` spec (concrete)

Tag taxonomy from `data/news.json` (verified): `정책` (286건), `기업` (1095건), `서비스` (1059건), `사고` (66건), `해외` (391건), `일반` (995건). 모든 기사는 `tags: string[]`을 가짐.

Score per item (D9 outside voice 반영, **hybrid tag × companies**):

1. **Tag priority** (multiplier): `정책` OR `사고` 포함 = ×3 · `기업` 포함 = ×2 · `서비스` 포함 = ×1.5 · 그 외 = ×1.
2. **Companies cardinality** (additive base): `1 + (item.companies ?? []).length`. `companies` field absent → base 1. **Match contract**: `companies[]` 항목은 PR12에서 normalize된 canonical `Company.name` 정확 일치 (e.g. "라이드플럭스 (RideFlux)"). 미래 cross-file validator로 더 강고화 가능 (Reviewer Concerns 참조).
3. **Summary length** (small bonus): `Math.min((item.summary ?? "").length, 500) / 1000`. 캡 500자. summary가 null/empty면 0. 0~0.5 boost.
4. **Headline localeCompare** (final tiebreaker): `Intl.Collator("ko", {sensitivity:"base"}).compare(a, b)` (Reviewer Concerns I 반영, 명시적 Collator로 ICU drift 방어).

```typescript
function scoreOf(item: NewsItem): number {
  const tagMult = item.tags.includes("정책") || item.tags.includes("사고") ? 3
    : item.tags.includes("기업") ? 2
    : item.tags.includes("서비스") ? 1.5
    : 1;
  const compBase = 1 + (item.companies ?? []).length;
  const summaryBoost = Math.min((item.summary ?? "").length, 500) / 1000;
  return tagMult * compBase + summaryBoost;
}

const collator = new Intl.Collator("ko", { sensitivity: "base" });
function pickRepresentative(items: NewsItem[]): NewsItem | null {
  if (items.length === 0) return null;
  return items.slice().sort((a, b) =>
    scoreOf(b) - scoreOf(a) || collator.compare(a.headline, b.headline)
  )[0];
}
```

**예시 비교** (광주 발표일 시나리오 outside voice 명시):
- 광주 발표 (기업 태그, 3개 회사): score = 2 × (1+3) + boost = **8.x**
- 동일 날 작은 사고 (사고 태그, 1개 회사 또는 0개): score = 3 × (1+1) + boost = **6.x**
- → 광주 발표 우세 (예상 직관과 일치). 사고 태그 단독으로는 큰 기업 발표를 누르지 못함. 강한 정책+다회사 조합만이 우세.

Deterministic. 같은 input → 같은 output. 새 기사가 들어와도 (publish 후 N분 늦게 도착) ranking 안정.

### KST 날짜 grouping (timezone correctness)

`published_at`은 UTC ISO. `[:10]` naive slice는 KST 늦은 저녁(예: KST 23:30 = UTC 14:30) → UTC 다음날로 bucket되는 버그. **Fix**: `groupByDate`는 KST 기준 날짜로 grouping. 의존성 추가 없이 (Intl 표준 사용):

```typescript
function toKSTDate(iso: string): string {
  // "2026-05-04T23:30:00+09:00" or UTC "2026-05-04T14:30:00Z" 둘 다 정상 처리
  const d = new Date(iso);
  // Intl로 KST 변환 후 YYYY-MM-DD
  const fmt = new Intl.DateTimeFormat("en-CA", { // en-CA gives YYYY-MM-DD
    timeZone: "Asia/Seoul", year: "numeric", month: "2-digit", day: "2-digit",
  });
  return fmt.format(d); // "2026-05-04"
}
```

`/archive/[date]` URL의 date는 항상 KST 기준. 사용자에게 "5월 4일 뉴스" = KST 5월 4일 0시 ~ 23:59:59. 한국 사용자 직관과 일치.

대안 평가:
- **Approach B (LLM augmentation)**: A + Claude API로 일별 2-3문장 요약. 더 좋은 narrative지만 API key handling + 환각 가드레일 + cron 1개 추가. 별도 follow-up PR으로 deferred.
- **Approach C (Storyline clustering)**: Threading + topic graph + heatmap. XL effort, 70% 버전이 사용자 더 헷갈리게 할 수 있음. Year-2 platform 플레이.

### Path conventions (locked)
- `/archive/[date]` where date = `YYYY-MM-DD` (e.g. `/archive/2026-05-04`).
- `/archive/[YYYY-MM]` where YYYY-MM (e.g. `/archive/2026-05`).
- **Implementation**: 단일 Next.js dir `app/archive/[slug]/page.tsx` 사용. URL 파라미터 `params.slug`를 regex dispatch:
  - `/^\d{4}-\d{2}-\d{2}$/` 매치 → DateView
  - `/^\d{4}-\d{2}$/` 매치 → MonthView
  - 그 외 → `notFound()`
- 두 개의 dir (`[date]/`, `[year-month]/`)을 두면 Next 라우터가 `/archive/2026-05` 같은 슬러그를 어느 dir에 매칭할지 ambiguous → 단일 `[slug]` dir로 통합.

## Reviewer Concerns (max iterations reached, 7/10 score)

Spec review loop hit 3-iteration cap. The following items raised in Round 3 are **acknowledged but deferred to implementation discretion** — engineer can address inline or kick to follow-up PR:

### MED severity
- **Sitemap `changeFrequency` for `today`/recent dates**: `monthly`는 immutable past dates에 맞음. 그러나 today's archive 페이지는 6h마다 mutate 가능 (새 기사 → 대표기사 swap). 권장 split: 최근 7일 → `daily`, 그 외 → `monthly`. Sitemap 빌드 로직에서 `(today - date) < 7days` 체크.
- **CalendarHeatmap a11y**: 0건 cell을 `<div>`로 wrapping하면 grid에서 keyboard tab 일관성 깨짐. 모두 `<a>` 유지하되 0건은 `aria-disabled="true" tabIndex={-1}` + `pointer-events-none`.
- **Build perf assumption**: module-level memoization은 Next.js worker 별로 작동, global 아님. 511 params를 N workers가 분산 빌드 시 worker 별로 corpus 재로드. Linear projection (1.6분) 가정. `next build` 실측 후 4분 초과 시 monthly route SSR fallback.

### LOW severity
- **`getDailyArchive()` sort O(N log N) × 511 pages**: module-level memoization으로 한 번만 계산되도록 강제 (lib level Map cache).
- **JSON-LD `datePublished`**: KST 자정 + offset 대신 `representative.published_at` 원본 timestamp 사용. Schema.org가 원하는 것은 publish time, not bucket day midnight.
- **`localeCompare("ko")` ICU determinism**: Node 마이너 업데이트 시 sort order drift 가능성. `Intl.Collator("ko", {sensitivity:"base"})` 명시 + 동률 재발생 시 final tiebreaker는 `item.url` byte compare로 강고함.
- **Cross-file canonical name validator extension**: 현재 `validate_data.py`는 `zones[].companies[]` cross-validation만 수행. `news[].companies[]`에도 동일 cross-check 추가하면 pickRepresentative의 company match가 강고해짐. `companies?` optional이라 strict 강제는 위험 (pre-backfill 기사 fail) → soft warning 형식 권장.

## Edge Cases (이미 보임)

- **빈 날 (zero articles)**: 코퍼스에 0건 날짜는 없음 (현재 436개 distinct days = 모두 ≥1건). `generateStaticParams`는 코퍼스에서 directly 추출 → 0건 날짜는 처음부터 포함 안 됨. 사용자가 임의 날짜 (예: `/archive/1999-01-01`) 직접 타이핑 → `dynamicParams: true`이지만 그 날 corpus에 없으면 페이지 코드 내 `notFound()` 명시 호출. 결과: 404. 예측 가능.
- **단일 기사 날 (1 article)**: 그 기사가 자동 대표기사. tiebreaker 안 돔. 정상. 그 날 페이지의 collapsible "전체 기사 N건"은 자동으로 안 보임 (1건 = 대표기사만 표시).
- **태그 없는 기사**: 모든 corpus 기사가 ≥1 tag 보유 (검증됨, 6 tags 분포 확인). 만약 미래에 무태그 기사 들어오면 → tag bucket D (0점), companies + summary로 결정.
- **태그 동순위 (정책 + 사고 둘 다 같은 날 있음)**: 둘 다 A-bucket. companies 다양도로 tiebreak. 그래도 동률 → summary 길이 → headline localeCompare(ko). 결정론적.
- **미래 published_at**: 크롤러 정상이면 발생 안 함. 만약 들어오면 generateStaticParams 포함. 정상.
- **2016-2024 historical sparse 데이터**: 하루 1-2건일 수 있음. 작은 날도 페이지는 만듦 — historical reference 가치 있음. heatmap에서는 1-3건 bucket으로 흐릿하게 표시.
- **WeeklyInsightWidget highlight null일 때**: 이미 `WeeklyInsight` 타입 spec상 `highlight: NewsItem | null`. null이면 link 자체 렌더 skip (현재 동작 유지).
- **`/archive/[date]` permalink 안정성**: date format은 ISO `YYYY-MM-DD` 영구 고정. 향후 변경 시 redirect 규칙 추가 (현재 시점엔 변경 계획 없음). Permalink 약속이 투자자 인용 가치의 핵심.
- **새 기사 도착 → 그 날 대표기사 변경**: 크롤러는 6h마다 동작. 새 기사가 더 높은 score 받으면 다음 빌드/revalidate에서 대표기사 swap. SEO 측면 약점은 아님 (헤드라인 페이지 콘텐츠는 아예 다시 올라가는 게 정상).
- **대량 페이지 빌드 시간**: 511 SSG pages × news.json 한번 로드 (`getAllNews()` module-level memoization, news.ts와 동일 패턴) → 빌드 시간 1분 → 약 1.5-2분 예상. 측정 필요. 만약 4분 넘으면 monthly만 dynamicParams=true SSR로 전환 (월 페이지는 SEO 우선순위 낮음).

## Eng Review Decisions (2026-05-11, D2-D13)

### Architecture (D2-D4)

**D2 — Build safety: bad `published_at` ISO can't kill 511-page route**
- `toKSTDate(iso)` returns `string | null` (was just `string`).
- Implementation: `if (!Number.isFinite(d.getTime())) return null;` before `Intl.DateTimeFormat.format`.
- `groupByDate(items)`: skip items where `toKSTDate` returns null.
- Effect: bad ISO → that item missing from archive (graceful degradation, /news pattern), not entire build failure.

**D3 — Freeze override at `getDailyArchive()` boundary, not inside `pickRepresentative`**
- `pickRepresentative` stays pure: `(items: NewsItem[]) => NewsItem | null`. No frozen.json read.
- `getDailyArchive()` does the override AT THE END: pick fresh per day → if `(today - date) >= 7d` AND date in frozen.json → swap fresh pick for frozen entry's NewsItem (lookup by frozen.url in current corpus).
- Pure function testable; freeze logic testable independently. Layer separation.

**D4 — JSON-LD `publisher` omit when source looks like a domain**
- Heuristic: `if (/\..+\./.test(item.source))` → omit publisher field (e.g. "v.daum.net", "naver.com").
- Else: `publisher: { "@type": "Organization", name: item.source }`.
- schema.org allows omit; better than false signaling.

### Code Quality (D5-D7)

**D5 — `toKSTDate` lives in `lib/news-utils.ts`**
- news-utils.ts owns "pure presentation/data utilities" (canonicalLink, formatRelativeKo, tagClass). toKSTDate fits.
- news-archive.ts imports from news-utils. Single direction; no circular dep.
- `computeWeeklyInsight()` can populate `highlight.date` without cross-module coupling.

**D6 — `WeeklyInsight.highlight` becomes discriminated union**
- Before: `highlight_headline: string | null` (3 separate nullable fields if D6 went path B).
- After:
  ```typescript
  highlight: {
    headline: string;
    date: string;       // KST YYYY-MM-DD
    url: string;        // canonicalLink(item)
  } | null;
  ```
- Single null check at consumer. Type enforces "all-three together or none".
- Migration: 1 line in WeeklyInsightWidget.tsx (`insight.highlight_headline` → `insight.highlight?.headline`).

**D7 — Frozen URL fallback observability**
- Runtime: `getDailyArchive()` warns when frozen URL missing from corpus: `console.warn('[news-archive] frozen rep missing for ${date} (${url}), falling back to fresh pick')`.
- CI: `validate_data.py` cross-checks every `frozen.json` URL exists in `news.json`. Missing → ERROR (not warning) + date+url details. Build fails until frozen.json cleaned up.

**D10 — Module-level `Intl.DateTimeFormat` singleton in news-utils.ts**
- ```typescript
  // Top of news-utils.ts:
  const KST_DATE_FORMATTER = new Intl.DateTimeFormat("en-CA", {
    timeZone: "Asia/Seoul",
    year: "numeric", month: "2-digit", day: "2-digit",
  });
  export function toKSTDate(iso: string): string | null {
    const d = new Date(iso);
    if (!Number.isFinite(d.getTime())) return null;
    return KST_DATE_FORMATTER.format(d);
  }
  ```
- Init once per module load (build-worker or serverless invocation). 3,090 items × .format() call ≈ 3ms total vs ~3s if instantiated per item.
