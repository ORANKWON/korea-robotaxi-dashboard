## Bundle 정의 (Approach B + 7 cherry-picks)

### Phase 1: 데이터 품질 (크롤러) — 2h CC
- `parse_title_source()` **end-anchored** 다중 구분자만 매칭: 마지막 ` - `, ` | `, ` · ` 뒤가 ≤30자 한글/영문/숫자만이면 source. `(...)` 패턴은 매칭 X (법안명/연도 false positive 방지).
- `infer_tags()` 키워드 대폭 확장:
  - **기업**: 우버/Uber, 딜리버리/Delivery, GM/Cruise, Zoox, Wayve, Aurora, Mobileye, Nvidia, Apollo, 바이두, 텐센트, 비야디/BYD, 샤오펑, 니오, 리오토 등
  - **정책**: 도로교통법, 자동차관리법, 운수사업법, 임시운행허가, 시범운행지구, 지정, 고시, 공포, 입법
  - **사고**: 사망, 부상, 추돌, 탑승자, 보행자, 신호위반, 무인 사고
  - **서비스**: 유료, 요금, 24시간, 심야, 무인, 원격
- `infer_companies()` 신규 함수 — Hangul-aware matching (no `\b`):
  - alias가 한글이면 단순 `in` substring + **부정 prefix/suffix 리스트** ("현대차" OK, "현대" 단독은 alias에서 제외)
  - alias가 라틴이면 `re.search(rf"(?<![A-Za-z0-9]){re.escape(alias)}(?![A-Za-z0-9])", headline, re.IGNORECASE)`
  - `companies: string[]` 필드 (회사 ID 또는 canonical name)
- **summary 재생성 (실제 작업)**:
  - description이 headline과 normalized-equal이면 → `fetch_meta_description()` (이미 있음, 미사용 케이스 활성화)
  - 그래도 실패하면 headline 첫 문장을 자르고 `…` 추가
  - backfill 시 696건 모두 재생성
- dedup threshold:
  - 기본 0.6 (현재 0.5보다 stricter — 더 엄격하게 dup으로 안 잡음 = 보수적)
  - **같은 source끼리는 0.45** (같은 매체 재게시 가능성 ↑ → 더 적극적으로 dup 잡음)
  - 코멘트에 "lower threshold = more aggressive dedup" 명시
- `unwrap_redirect_url()` 신규 — Google News는 base64 디코딩 필요:
  - `CBMi...` payload는 base64로 publisher URL 인코딩됨
  - `googlenewsdecoder` 라이브러리 (PyPI) 사용 또는 자체 구현 (디코더 코드 100라인 이내)
  - 실패 시 원본 URL 유지, 로그
  - 성공률 목표 70%+ (현실적 — 100%는 Google이 막을 수 있음)
- `companies.json` alias 필드 추가:
  ```json
  { "name": "모셔널 (현대차그룹)", "aliases": ["모셔널", "Motional", "현대모셔널"] }
  ```
- 기존 696건 backfill 스크립트 `backfill_news.py`:
  - 단계: re-parse title → re-infer tags/companies → re-extract summary → unwrap URL
  - asyncio + Semaphore(10) 동시성 (HEAD/decoder 병렬)
  - dry-run 모드 (`--dry-run`) — 결과만 print, 파일 미수정
  - 50건 sample 비교 후 본 실행 (regression 방지)

### Phase 2: 공개 데이터 (API + RSS) — 0.75h CC

**Data contract:**
| Endpoint | Params | Response | Cache |
|----------|--------|----------|-------|
| `/api/news.json` | `?tag=`, `?company=`, `?q=`, `?limit=` (default 100, max 500), `?offset=` | `{items: NewsItem[], total: number, limit, offset}`, sort: `published_at` desc | ISR 6h (정렬 = 크롤 주기) + `s-maxage=21600` |
| `/api/news.csv` | 동일 | RFC 4180 CSV, BOM, CRLF, all 696 if no params | ISR 6h, `Content-Type: text/csv; charset=utf-8` |
| `/feed.xml` | 없음 | RSS 2.0, **최근 50건** (reader 표준), `final_url` 사용, 태그 → `<category>` | ISR 6h |

- `eTag` = sha256(news.json) 첫 8글자 → `If-None-Match` 304 지원
- ISR 1h가 아닌 **ISR 6h** = 크롤 cadence와 일치 (낭비 revalidation 제거)
- 입력 검증: `q` max 200자, `limit` 1-500, `offset` >=0, 잘못된 값은 400 또는 default fallback

### Phase 3: /news 단독 페이지 — 1.5h CC
- `/news` route — **SSR + ISR(6h)** (SSG는 `?page`/`?q`/`?company` 조합 폭발로 부적합). search params는 페이지 내부 `'use client'` boundary에서 처리. sitemap에는 `/news` 하나만 등록 (필터 조합 X)
- `<NewsHeader />` — title + RSS subscribe button + JSON/CSV 다운로드
- `<WeeklyInsightWidget />` (cherry-pick #7) — 지난 7일 집계
  - Top 3 most-mentioned companies (`companies` 필드 카운트, 헤드라인 재스캔 X — Phase 1의 infer_companies 결과 재사용)
  - 주요 이벤트 1행 (정책/사고 태그 첫 헤드라인)
  - 스파크라인 (recharts LineChart, 일별 기사 수)
  - Empty state ("데이터가 쌓이지 않았습니다")
- `<NewsFiltersBar />` — 태그 chips + 기업 chips + 검색 input
- `<NewsList />` — 페이지네이션 50/페이지, sort = published_at desc (default), unread-first 토글 옵션. 696/50 = 14페이지. 페이지 컨트롤: `<Prev | 1 2 … 14 | Next>`
- `<NewsCard />` — 헤드라인 + summary 2줄 + source · 상대시간 · 태그 chips + 북마크/읽음 토글
- `<RelatedNews companyId={id} limit={10} />` (Phase 5에서 사용) — `companies` 필드를 직접 사용 (헤드라인 재스캔 X)

### Phase 4: 인터랙션 (검색/필터/URL/읽음/북마크) — 1.5h CC
- `useSearchParams()` Suspense wrapper component
- 모든 state가 URL에 영속화: `?q=`, `?tag=`, `?company=`, `?page=`
- `useLocalStorage()` SSR-safe hook (typeof window check)
- `bookmarks: Set<string>` (URL 키), `readUrls: Set<string>` (URL 키)
- 카드 우클릭 / 별 아이콘 → bookmark
- 카드 클릭 → 자동 read 마킹 + opacity-60
- "읽은 거 숨기기" 토글

### Phase 5: 통합 — 0.5h CC
- 홈 `<NewsFeed />` → `/news` 페이지의 simple variant 재사용 (top 10만, 검색/페이지네이션 없음)
- `/company/[id]`에 `<RelatedNews companyId={id} limit={10} />` 추가 + "더보기 → /news?company=ID"
- `sitemap.ts` `/news` 추가
- `app/layout.tsx` 헤더 nav에 `/news` 링크

## Edge Cases

| 상황 | 처리 |
|------|------|
| `useSearchParams()` SSR | Suspense boundary로 감싸기 |
| localStorage = undefined (SSR) | typeof window check + null fallback |
| WeeklyInsight 데이터 < 7일 | "데이터가 쌓이지 않았습니다" empty state |
| redirect unwrap 실패 (Google이 차단) | 원본 URL 유지, 다음 크롤 때 재시도 |
| 검색어 빈 문자열 | 필터링 건너뜀, 전체 표시 |
| 한글 IME 조합 중 입력 | `onCompositionEnd` 사용 (매 키스트로크 검색 X) + 200ms debounce |
| 페이지 번호 > 최대 | 빈 결과 + 1페이지 링크 안내 |
| 필터 조합 결과 0건 | "조건에 맞는 뉴스가 없습니다" + 필터 초기화 버튼 |
| 동시 두 탭 북마크 토글 | `storage` 이벤트 리스너로 동기화 |
| 태그 chip + 기업 chip 둘 다 선택 | AND 조건 (둘 다 만족) |
| CSV에 큰따옴표/줄바꿈/콤마 | RFC 4180: `"` 필드 wrapping + `""` escaping, CRLF |
| RSS reader가 절대 URL 요구 | site origin prepend (`https://korea-robotaxi-dashboard.vercel.app`) |
| 696건 backfill 중 크롤러 동시 실행 | 락 파일 또는 순차 실행 권장 (cron 정지 후 backfill) |
| 기업 alias 충돌 ("현대" → 현대차/현대모비스) | 정확한 alias만 (`현대차`, `현대자동차`, 단독 "현대" 제외) |
| /api/news.csv 요청 폭주 | ISR 6h (Phase 2 contract와 일치) + eTag 304 |
| `final_url` 필드 없는 기존 기사 | UI fallback: `final_url || url` |
